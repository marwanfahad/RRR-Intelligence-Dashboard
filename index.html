How this is calculated
For each region (Region2 canonicalised via normalizeRegion()), we take the last 6 weeks of RRR% and volume for that region only:
  • Expected = weighted-moving-average of last 6 weeks (60% EWMA + 40% linear projection), same method the national forecast uses
  • Confidence band = Expected × 0.85 (Low) and Expected × 1.15 (High), a ±15% envelope
  • Direction = ↑ if Wk+1 > last week by ≥ 1pp, ↓ if lower by ≥ 1pp, → otherwise
  • Regions with fewer than 3 recent weeks show insufficient history instead of a forecast

🧩 FRU Journey — Model × Symptom → Resolving FRU
Scans the entire dataset. For each Sub-Series (ranked by repeat volume), pick a symptom to see the First WO FRU# that was sent vs the Next SO FRU# that actually resolved the repeat. Excludes DOA.
# Sub Series (Model) Top Symptom (highest RRR) Recommended Fix FRU#
How this is calculated: For every repeat-flag WO (FLAG=1, excluding DOA) where the first-visit symptom loosely matches the next-visit symptom, we take the First WO FRU# (what was sent originally) and the Next SO FRU# (what was sent on the second WO that closed the case). Rows where the two FRUs are identical or blank are excluded — those don't teach us anything. Volumes beside model / symptom / FRU show repeat hits tied to that combination, so you can size the opportunity: “sending FRU X first for Model Y × Symptom Z avoids N repeats”.
